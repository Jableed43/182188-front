===== SLIDE 5 =====
Bloque 1 — ¿Qué es CSS?
Cascading Style Sheets: lenguaje de diseño que da instrucciones visuales (color, tamaño, posición).
Hojas de estilo: archivo .css con las reglas de belleza.
Cascada: las reglas fluyen de arriba hacia abajo; ante conflictos, gana una jerarquía estricta.
Anatomía de una regla

Bloque 2 — ¿Dónde escribo CSS?
Inline (style="..."): PROHIBIDO, ensucia el HTML.
Interno (<style>): solo pruebas rápidas.
Externo (.css + <link>): el estándar profesional.

===== SLIDE 15 =====
Bloque 3 — Selectores
Etiqueta: p { } — bombardeo masivo (todos los <p>).
Clase (.): .mi-boton { } — el recomendado, ataque de precisión.
ID (#): #header { } — único por página (como un DNI).

===== SLIDE 25 =====
Herencia: ¿qué pasa a los hijos?
SÍ heredan: color, font-family, text-align (texto). Forzable con inherit.
NO heredan: border, padding, margin, background, width, height (el Box Model).

===== SLIDE 27 =====
Bloque 5 — El Modelo de Caja (Box Model)
TODO elemento es una caja. Analogía del cuadro en una galería:
Content: la obra (texto/foto).
Padding: el paspartú (espacio interno).
Border: el marco.
Margin: distancia en la pared (espacio externo).
Margin vs Padding
Curso Front-End  /  Módulo 1: Arquitectura y Maquetación Moderna  /  Unidad 05  /  Introducción a CSS: Estilo y Cascada

===== SLIDE 28 =====
Bloque 5 — El Modelo de Caja (Box Model)
Curso Front-End  /  Módulo 1: Arquitectura y Maquetación Moderna  /  Unidad 05  /  Introducción a CSS: Estilo y Cascada
Característica | padding (interno) | margin (externo)
Ubicación | Dentro del borde | Fuera del borde
Color de fondo | Toma el de la caja | Transparente
Función | El contenido "respira" | Separa cajas vecinas
